# 百变投稿版输出规范

项目会在每次生成结果目录下自动创建 `submission/`，并生成同级投稿压缩包：

```text
剧本名+人数+题材+作者微信号.zip
```

## 目录结构

```text
submission/
  00_投稿版目录说明.md
  00_submission_manifest.json
  01_简介、角色、区域.docx
  02_规则说明.docx
  03_亮点、作者阐述.docx
  04_结局.docx
  人物剧本/
    角色名_第一幕人物剧本.docx
  线索/
    01_第一幕线索.docx
    99_道具文本与特殊线索附件.docx
  地图/
    01_地图与房间示意图需求.docx
    README_地图补图说明.txt
  投票/
    01_第一幕投票.docx
  真相/
    01_第一幕真相.docx
    02_故事还原.docx
```

## 自动满足的要求

- 输出 Word 文档 `.docx`，正文使用宋体并设置首行缩进。
- 人物剧本按角色拆分，文件名包含角色名和幕次。
- 线索按幕/轮拆分为 Word 表格，包含【搜证地点】【线索名】【线索内容】【备注】。
- 投票文档按角色生成题目，并用红字标记正确选项。
- 真相文档包含故事还原和玩家视角推理链草案。
- 自动生成投稿版压缩包，便于后续补作者微信号后发送。

## 仍需人工复核/补充

- 投稿邮箱主题中的作者微信号。
- 地图图片或手绘草稿图，尤其是含推理必需信息的地图。
- 投票题目是否与每个角色当幕任务强关联。
- 每个角色投票题数量是否符合最终设计。
- 红字正确选项是否准确。
- 推理链是否完全站在玩家视角，而不是上帝视角。
- 内容原创性、是否疑似撞梗、是否符合平台最新要求。
